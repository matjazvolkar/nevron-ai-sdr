0 Domain purchase & pre-flight

Registrar

Recommended: Cloudflare Registrar (at-cost pricing, free DNS, DNSSEC one-click, no upsells). Backup: Namecheap, Porkbun.

Avoid: GoDaddy (upsell hell, slower DNS propagation).

Purchase checklist

  • Registration period: minimum 2 years — 1-year is a spammer signal.
  • WHOIS privacy: enable (Cloudflare does it by default) but use a legitimate registrant org when asked.
  • Auto-renew: ON — a domain that expires mid-campaign = dead accounts.
  • Registrant info: match other Nevron-owned domains for consistency.

Wait before sending

Do not send before Day 30 Fresh domains are filtered aggressively by Gmail/Outlook regardless of DNS setup. 30 days minimum, 90 preferred.

Nameservers

If you use Cloudflare DNS (recommended — fast propagation, clean UI), point registrar nameservers to:

ella.ns.cloudflare.com
walt.ns.cloudflare.com

(Your actual pair is shown in the Cloudflare dashboard when you add the domain.)

1 Google Workspace setup

Plan

Business Starter ($6/user/month) is sufficient — you only need mailboxes, not Drive/Meet enterprise features. Scale to Standard only if you need 2TB+ per user.

Add domain to existing Workspace

  1. Google Admin → Account → Domains → Manage domains
  2. "Add a domain" → enter trynevron.com → "Secondary domain" (users in primary workspace can receive @trynevron.com).
  3. Verify via DNS TXT — Google gives you a record like google-site-verification=abc123... — publish it at root (@).

Create mailboxes — 6-persona roster

Same six real employees on every new outreach domain. Each person keeps a matching mailbox across every Nevron outreach domain — same name, same LinkedIn URL, same signature everywhere.

Per-mailbox post-creation checklist

  • Full display name (e.g. "Rok Kokalj")
  • Profile photo uploaded in Gmail settings
  • Real signature: name + role + Nevron + LinkedIn URL
  • 2FA enabled
  • App password generated (for Instantly OAuth — or use OAuth directly if Instantly supports)
  • Login to Gmail web UI at least weekly going forward

2 Core authentication (MX / SPF / DKIM / DMARC)

MX records — route mail to Google

Google Workspace now uses a single MX record (modernized from the old 5-record set):

Type   Name   Priority   Value
MX     @      1          SMTP.GOOGLE.COM.

Delete any existing MX records first — mixed MX = broken mail.

SPF — a single TXT record

Type   Name   Value
TXT    @      v=spf1 include:_spf.google.com ~all
Only ONE SPF record at root Two SPF records = automatic SPF failure → DMARC failure → inbox placement dies. If you later add Instantly / SendGrid / etc. to the send path, merge them into this single record.

Upgrade after 2 weeks clean

Once DMARC reports show no legit senders failing, tighten the policy:

v=spf1 include:_spf.google.com -all

-all (hard fail) > ~all (soft fail).

DKIM — 2048-bit, generated in Google Admin

  1. Google Admin → Apps → Google Workspace → Gmail → Authenticate email
  2. Select domain trynevron.com
  3. Generate new record → DKIM key length: 2048 bit (never 1024) → Selector: google (default).
  4. Google shows the TXT record to publish:
Type   Name                    Value
TXT    google._domainkey       v=DKIM1; k=rsa; p=MIIBIjANBgkq... (long string)
  1. Publish the record in DNS → wait 10-30 minutes for propagation.
  2. Return to Admin → "Start authentication". Status flips to "Authenticating email" → "Authenticated" within an hour.

DMARC — phased rollout over 6 weeks

Create a dedicated inbox for aggregate reports first: dmarc@trynevron.com (or forward to Dmarcian's inbox — see Phase 6).

Phase 2a — Monitor only (Week 1-2)

Type   Name      Value
TXT    _dmarc    v=DMARC1; p=none; rua=mailto:dmarc@trynevron.com; pct=100; adkim=s; aspf=s; fo=1

Phase 2b — Quarantine (Week 3-4, after reports are clean)

v=DMARC1; p=quarantine; rua=mailto:dmarc@trynevron.com; pct=100; adkim=s; aspf=s; fo=1

Phase 2c — Reject (Week 5+, full enforcement)

v=DMARC1; p=reject; rua=mailto:dmarc@trynevron.com; pct=100; adkim=s; aspf=s; fo=1
Don't skip phases Jumping straight to p=reject can silently drop legit mail (e.g., Google Calendar invites, Zoom notifications forwarded via the domain). Each phase is a checkpoint.

3 Web presence — 301 redirect to nevron.co

A cold-email domain that returns ECONNREFUSED is a textbook spam signal. The cheapest robust fix: 301 redirect to the real Nevron website.

Option A — Cloudflare Pages / Worker (recommended, free)

  1. Cloudflare dashboard → Rules → Redirect Rules → Create rule
  2. Name: trynevron.com → nevron.co
  3. When incoming requests match: hostname equals trynevron.com OR hostname equals www.trynevron.com
  4. Then: Static redirect → 301 Permanent Redirect → Target URL: https://nevron.co$1 (preserves path).
  5. DNS: proxied A record at root pointing to 192.0.2.1 (any dummy — Cloudflare intercepts via proxy).

Option B — Registrar-level URL forwarding

Namecheap, Porkbun, Cloudflare all offer "URL forwarding" — simpler, one click. Confirm it's 301 permanent (not 302 temporary, not frame forwarding).

Verify the redirect

curl -I https://trynevron.com
# Expect:
# HTTP/2 301
# location: https://nevron.co/

SSL

Let's Encrypt via Cloudflare or your host — free, auto-renew. A broken cert on the redirect host is as bad as no redirect.

Bonus: www record Add CNAME www @ so www.trynevron.com also redirects. Users occasionally type it that way.

4 Advanced DNS (MTA-STS / TLS-RPT / DNSSEC)

MTA-STS — force TLS for inbound mail

Requires both a DNS record AND an HTTPS-hosted policy file.

DNS record

Type   Name                 Value
TXT    _mta-sts             v=STSv1; id=2026042100

(The id is a version string — change whenever you update the policy file.)

Policy file

Host at https://mta-sts.trynevron.com/.well-known/mta-sts.txt:

version: STSv1
mode: enforce
mx: *.google.com
mx: aspmx.l.google.com
mx: *.googlemail.com
max_age: 604800

Easiest: Cloudflare Pages with a static file, or GitHub Pages with a CNAME for mta-sts.trynevron.com.

TLS-RPT — TLS failure reports

Type   Name               Value
TXT    _smtp._tls         v=TLSRPTv1; rua=mailto:tlsrpt@trynevron.com

DNSSEC

At Cloudflare Registrar / most modern registrars: Domain → DNSSEC → Enable. One click. Confirm with dig trynevron.com +dnssec +short showing RRSIG records.

BIMI (optional, costs money)

Requires DMARC at p=quarantine+ AND a Verified Mark Certificate (VMC) — about $1,500/year from DigiCert or Entrust. Only worth it if Nevron's logo is trademark-registered and volume is 10K+/month. Skip for now.

5 Instantly custom tracking domain

Instantly's default tracking domain is shared across all customers — poisoned by other people's spam. Always use your own.

Setup

  1. Instantly → Settings → Custom Tracking Domain → Add domain
  2. Instantly gives you a CNAME target like open.sleadtrack.com
  3. In Cloudflare DNS, add:
Type    Name     Target                   Proxy
CNAME   track    open.sleadtrack.com      DNS only (unproxied, gray cloud)
DNS only — never proxied If you leave the orange Cloudflare cloud ON, Instantly's tracking breaks. Grey cloud is mandatory for this record.
  1. Back in Instantly → click "Verify" — status flips to green within 10 minutes.
  2. Assign track.trynevron.com to all campaigns sending from this domain.

Open tracking — turn it OFF

In 2026, Apple Mail Privacy Protection pre-fetches pixels → fake opens. Track links only. This also reduces the number of tracking-related elements in your emails = better deliverability.

6 Monitoring (Postmaster / Dmarcian / Mail-Tester)

Google Postmaster Tools

  1. postmaster.google.com → Add domain trynevron.com.
  2. Verify via DNS TXT (Google provides the record).
  3. Wait 1-2 weeks after first real sends for data to populate.
  4. Watch: Spam rate < 0.1%, Domain reputation "High", IP reputation "High", Authentication 100%.

Dmarcian — free DMARC report parser

  1. Sign up at dmarcian.com — free tier covers up to 5K messages/day.
  2. They give you an RUA address like xyz-trynevron@rua.dmarcian.com.
  3. Update your DMARC record to forward reports there:
v=DMARC1; p=none; rua=mailto:dmarc@trynevron.com,mailto:xyz-trynevron@rua.dmarcian.com; pct=100

Dmarcian parses the daily XML blobs into a readable dashboard. Required for moving safely from p=none → quarantine → reject.

Mail-Tester — pre-launch sanity check

  1. Visit mail-tester.com, get a throwaway address.
  2. Send one email from each new mailbox (nina.bernik, amadej.demsar) to that address.
  3. Score should be 10/10. Anything below 9 → fix before first real send.
  4. Repeat monthly as a drift check.

MXToolbox — blacklist monitoring

Manual spot-check at mxtoolbox.com/blacklists.aspx — enter trynevron.com. Paid tier does this automatically with alerts.

7 Warmup & first send

Connect mailboxes to Instantly

Per mailbox:

  • Instantly → Email Accounts → Add → Google OAuth
  • Settings:
daily_limit:         33
warmup.limit:        10
enable_slow_ramp:    true
weekday_only:        true
sending_gap:         1   (minute between sends)

Warmup duration

Week 1 — Pure warmup

~10 warmup messages/day. Zero real sends. Watch warmup health in Instantly → should hit 90%+ by end of week.

Week 2 — Warmup + small ramp

Warmup continues. Real sends: 5-10/day, soft audience (LinkedIn 1st-degree, known contacts).

Week 3 — Gradual ramp

Real sends: 15-25/day. Still warm audiences preferred.

Week 4+ — Full production

33/day per account to cold ICP leads. Warmup stays ON forever in the background.

Infrastructure target trynevron.com contributes 6 × 33 = 198/day at full ramp. Running parallel to nevron.net's 6 accounts (also 198/day) = 396/day total across 2 domains. Per-domain economics, full N-domain scale table, and the 18-month ramp plan live in the cold outreach scaling strategy.
